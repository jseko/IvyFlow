# Privacy Notice — IvyFlow

**Last updated: 2026-06-17**

IvyFlow (`ivyflow-cli`) is a **local-only** tool. It does not phone home, collect telemetry, or send data over the network.

## Data storage

All data generated by IvyFlow is stored in the `.ivy/` directory at your project root:

| Path | Contents | Scope |
|---|---|---|
| `.ivy/project.yaml` | Platform configuration, installed platforms, analytics toggle | Configuration |
| `.ivy/sessions/raw/events.jsonl` | L1 raw events (git commits, phase transitions, file_save events) | Local analytics |
| `.ivy/sessions/raw/checksums.json` | L1 deduplication index | Local analytics |
| `.ivy/sessions/inferred/sessions.jsonl` | L2 inferred sessions (heuristic boundaries) | Local analytics |
| `.ivy/sessions/cache/` | Derived cache: trend profiles, phase duration stats, calibration profile, suggestion feedback, rule manifest, adoption profile | Local cache |
| `.ivy/reports/` | Export: HTML dashboard reports (on demand) | Export |

## File Path Handling

IvyFlow records file paths relative to the repository root only (e.g., `src/core/main.ts`). Absolute paths are never stored. File content is never read or stored — per §9.11 evolution constraint, the Git Watch feature emits metadata only (path, status, commit hash, file size, insertions/deletions).

## Calibration Cache

The `ivy suggest --calibrate` command writes threshold calibration data to `.ivy/sessions/cache/calibration_profile.json`. This is Derived Cache — it stores computed P80 percentile values and calibration metadata (version, mode, timestamp). Per §9.12, calibration never modifies L1 (raw events) or L2 (inferred sessions).

## Rule Governance Cache

The `ivy rules --override` command writes user overrides to `.ivy/sessions/cache/rule_manifest.json`. This is Derived Cache — it stores effective rule configurations with applied overrides. Per §9.14, overrides never mutate built-in rule defaults through shared state. `ivy explain` reads this cache to display trace information but never writes data (per §9.15, read-only).

## Adoption Analytics

The `ivy analytics` command computes an adoption profile from existing L1 events (git_commit, phase_transition) and caches it at `.ivy/sessions/cache/adoption_profile.json`. Per §9.13, all metrics carry explicit confidence annotations and are descriptive statistics only — no predictive or causal claims. Analytics is based on data already recorded by git hooks; it does not collect new data.

## Suggestion Feedback

The `ivy suggest --mark-resolved` and `ivy review` commands record feedback in `.ivy/sessions/cache/suggestion_feedback.json`. This data is:

- **Stored only on your local machine**
- **Never transmitted anywhere**
- **Used solely to compute quality metrics** (effectiveness, accuracy, per-type acceptance rates) displayed via `ivy suggest --quality`, `ivy suggest --json`, and `ivy dashboard --quality`

You can delete any IvyFlow data at any time with `ivy uninstall`.

## What IvyFlow does NOT do

- Does **not** collect telemetry or usage statistics
- Does **not** send data to any server
- Does **not** read file contents (filenames only, relative paths)
- Does **not** embed tracking pixels, analytics SDKs, or third-party scripts
- Does **not** require an internet connection (except `ivy update` for version checks, and `ivy analytics` when GitNexus overlay is enabled)
- Does **not** record AI model prompts, responses, or tool invocations

## Git hooks

IvyFlow installs two git hooks:

1. **pre-push** (`.git/hooks/pre-push`): blocks pushes from non-archive changes. Also runs non-blocking `ivy check` reminder. Wrapped with IvyFlow markers for clean uninstall.
2. **post-commit** (`.git/hooks/post-commit`): v2 — records commit events and optional file_save events (when `IVY_GIT_WATCH=1`) to `.ivy/sessions/raw/events.jsonl`. Only active when `analytics_enabled: true` in `.ivy/project.yaml`.

Both hooks are local-only. They inspect git state within your repository and write to `.ivy/`. No data leaves your machine.

## Deletion

Run `ivy uninstall` to:

1. Remove all IvyFlow files from each configured platform
2. Surgically remove the git pre-push hook (preserving any custom content)
3. Delete `.ivy/` directory contents

Use `ivy uninstall --dry-run` to preview what will be removed without making changes. To fully remove all cached data: `rm -rf .ivy/`.
