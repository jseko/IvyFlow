# Spec Quality Check — 10-Item Checklist

Run this checklist before confirming the proposal phase. Each item must be explicitly addressed.

| # | Check | What to Look For |
|---|-------|-----------------|
| 1 | Problem statement clear | Can a new team member understand what problem this solves in 30 seconds? |
| 2 | Scope bounded | Are explicit non-goals listed? What is NOT being changed? |
| 3 | Tech constraints identified | Language, framework, database, deployment target stated? |
| 4 | Business constraints identified | Regulatory requirements, SLA targets, data privacy rules? |
| 5 | Process constraints identified | Code review requirements, testing standards, deployment freeze windows? |
| 6 | Data constraints identified | Schema changes, migration strategy, backward compatibility? |
| 7 | Acceptance criteria testable | Can each criterion be verified with a yes/no check? |
| 8 | No ambiguous terms | Words like "fast", "scalable", "robust" replaced with specific metrics? |
| 9 | Tasks granularity 30-90 min | Each task completable in one focused session? |
| 10 | No silent assumptions | Did the AI add features, defaults, or behaviors not explicitly requested? |
