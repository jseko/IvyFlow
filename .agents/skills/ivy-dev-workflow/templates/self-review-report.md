# Spec Self-Review 报告模板

> 本模板供步骤一 1.10 Spec Self-Review 使用，标准化自检报告格式。

---

## 报告格式

```markdown
# Spec Self-Review — {提案名称}

> 生成时间：{时间戳}  
> 执行模式：{EXECUTION_MODE}  
> 检查文档：proposal.md / design.md / specs/**/*.md / tasks.md

## P0 Placeholder 扫描

| 状态 | 文件 | 位置 | 发现 | 修复动作 |
|------|------|------|------|---------|
| FIXED | design.md | §性能目标 | "TBD" | 替换为确认问题 |
| NEEDS_USER_INPUT | specs/auth/spec.md | §错误码 | "待补充" | 无法推断，需用户确认 |

**结果**：{通过数} 项通过 / {修复数} 项自动修复 / {待确认数} 项需用户确认

## P1 内部一致性检查

| 状态 | 检查项 | 发现 |
|------|--------|------|
| PASS | proposal vs design 功能范围 | 一致 |
| WARN | tasks.md 任务数与 design §实现计划 | 任务数 {X} ≠ 计划数 {Y} |

**结果**：{通过数} 项通过 / {警告数} 项警告

## P2 格式完整性

| 状态 | 文件 | 检查项 | 发现 |
|------|------|--------|------|
| PASS | proposal.md | 包含必要章节 | §背景、§功能范围、§验收标准 俱全 |
| FAIL | tasks.md | 所有任务有文件路径 | 任务 #{N} 缺少 File 标注 |

**结果**：{通过数} 项通过 / {失败数} 项缺失

## 汇总

| 级别 | 通过 | 修复 | 警告 | 待确认 |
|------|:---:|:---:|:---:|:---:|
| P0 | {n} | {n} | — | {n} |
| P1 | {n} | — | {n} | — |
| P2 | {n} | — | {n} | — |
```

---

## 状态标注说明

| 标注 | 含义 |
|------|------|
| PASS | 检查通过，无问题 |
| FIXED | 发现问题并自动修复（附修复动作） |
| WARN | 发现不一致但不确定是否为问题，需用户判断 |
| FAIL | 格式不完整，需用户补充 |
| NEEDS_USER_INPUT | 发现问题但无法自动修复，需用户决策 |

---

## 使用规则

1. P0 发现 Placeholder → 先尝试自动修复，无法修复的标注 NEEDS_USER_INPUT
2. P1 发现不一致 → 标注 WARN，不自动修改文档，由用户在步骤二决定
3. P2 发现格式缺失 → 标注 FAIL，提示具体缺失项
4. 报告写入 `openspec/changes/{提案名称}/self-review.md`
5. 报告摘要附在步骤二确认界面上，完整详情见文件
