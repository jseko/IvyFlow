# ADR-001: 需求分析不用独立 Agent

**状态：** 已采纳
**日期：** 2026-05-29
**决策者：** AI (评审优化)

## 上下文

在 v3.2 设计中，需要决定需求分析阶段是使用独立的 Agent 执行，还是由主 Agent 在对话上下文中直接执行。

需求分析的核心输入是用户与 AI 的对话内容。如果使用独立 Agent，需要将对话内容二次传递给该 Agent，可能会丢失微妙的上下文信息（如用户的隐含意图、之前的对话纠正等）。

## 决策

**需求分析由主 Agent 直接执行，不使用独立 Agent。**

需求分析作为步骤一的前半部分（1.1-1.3），由主 Agent 利用对话上下文直接完成：
- 提案名称生成（kebab-case 规则）
- proposal.md 填充
- 业务流程图绘制

## 选项对比

| 选项 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| 独立 Agent | 职责分离，可复用 | 两次传递丢失微妙上下文，增加调用成本 | ❌ |
| 主 Agent | 信息保真，零额外 Token 成本 | 职责边界不如独立 Agent 清晰 | ✅ 采纳 |

## 后果

**正面**：
- 需求解析结果与用户原始意图高度一致
- 减少一次 Agent 调用的 Token 消耗
- 用户可以在同一次对话中直接纠正需求理解偏差

**负面**：
- 主 Agent 职责范围略大（但仍在可管理范围内）
- 无法将需求分析能力独立复用到其他场景

**注意事项**：
- 需求分析阶段的边界通过 §4 章节明确声明
- 会议纪要等外部文档的解析建议由用户预处理后提供
