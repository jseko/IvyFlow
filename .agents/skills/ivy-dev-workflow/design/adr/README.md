## 什么是 ADR

ADR（Architecture Decision Record）是一种轻量级的架构决策文档，记录：
- 做过什么决策
- 为什么这么做
- 有哪些替代方案
- 决策带来的影响

## ADR 格式

每个 ADR 文件遵循统一格式：

```markdown
# ADR-00X: {决策标题}

**状态：** 已采纳 / 已废弃 / 已替代
**日期：** YYYY-MM-DD
**决策者：** AI / 人工

## 上下文
{问题描述，为什么需要做决策}

## 决策
{最终选择}

## 选项对比
| 选项 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| ... | ... | ... | ... |

## 后果
{正面影响 / 负面影响 / 注意事项}
```

## 维护方式

- ADR 为一次性写入的静态文档，后续不自动更新
- 新决策时在此目录创建新的 ADR 文件
- 废弃的 ADR 将状态改为"已废弃"并说明替代者
