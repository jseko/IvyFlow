# Design.md 生成指南

> 从 SKILL.md 步骤 1.4 提取：分级填充策略和章节详情。

---

## 分级填充策略（减少 token 消耗）

| 优先级 | 章节 | 填充策略 | 预计 Token |
|--------|------|----------|-----------|
| **P0** | 1, 2, 4, 5, 13 | 完整填充，包含图表和代码示例 | ~4000 |
| **P1** | 3, 6, 7 | 生成关键图表，简要说明 | ~2000 |
| **P2** | 8-12, 14+ | 引用项目现有规范或使用默认模板 | ~500 |

---

## P0 章节（必须详细填充）

- 第 1 章（概述）：从 proposal.md 提取
- 第 2 章（项目对齐）：使用核心变量填充技术栈、编码规范、包结构
- 第 4 章（数据模型）：设计 ER 图、表结构、实体类、DTO
- 第 5 章（API 设计）：设计接口列表、请求/响应示例
- 第 13 章（实施计划）：分阶段拆解任务（用于生成 tasks.md）

## P1 章节（中等详细）

- 第 3 章（架构设计）：绘制模块关系图（Mermaid）
- 第 6 章（前端设计）：设计页面路由、组件拆分（如有前端）
- 第 7 章（关键流程）：绘制时序图或状态流转图

## P2 章节（引用现有规范）

- 第 8-12 章：引用项目现有的安全、错误、性能、测试规范
- 第 14 章及以上：使用模板默认内容，只修改关键字段

## 文件写入错误处理

如果在生成 design.md、tasks.md 或其他大文件时遇到 `Error writing file` 错误：

1. **分段写入**：首次使用 Write 工具创建文件（前 100-150 行），后续用 Edit 工具每次追加 100-150 行，重复直到完整
2. **错误恢复**：Write 失败后立即切换到分段写入策略，所有必要信息（需求、提案名称、技术栈）已在前置步骤中确定
3. **禁止**：不要一次性写入超过 200 行的大文件；不要在 Write 失败后重复调用 Write；不要跳过文件生成步骤
