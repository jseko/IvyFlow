# 知识库

> 本目录存放 ivy-dev-workflow 的项目知识，按类型分层存储。
> 创建时间：2026-05-29（v3.2）

---

## 目录结构

| 目录 | 用途 | 填充方式 |
|------|------|---------|
| `business/` | 业务领域知识 | 按项目迭代逐步填充 |
| `dev/` | 开发技术知识（命名规范、编码约定等） | 初始化骨架，用户扩展 |
| `topology/` | 系统拓扑知识（服务拓扑、模块依赖、接口契约） | 架构变更时更新 |
| `memory/` | 记忆引擎（历史实现摘要、决策经验） | 每次归档时自动追加 |

## 知识库维护

- **business/** 和 **topology/** 由用户根据项目实际情况手动维护
- **dev/** 中的 `naming-conventions.md` 提供命名规范骨架，用户可根据团队约定修改
- **memory/** 由 skill 自动管理，用户无需手动编辑（检索入口为 `INDEX.md`）

## 与记忆引擎的关系

- `knowledge/memory/INDEX.md` 是记忆引擎的检索入口
- 每次通过 ivy-dev-workflow 完成功能实现并归档时，自动在 INDEX.md 追加摘要行
- 过期记忆（6 个月未访问）自动移至 `memory/archive/`

## 检索机制

- 用户需求中出现技术标签时自动检索 INDEX.md
- 检索阈值：≥ 2 个标签匹配
- 检索结果用于辅助需求分析和方案设计
