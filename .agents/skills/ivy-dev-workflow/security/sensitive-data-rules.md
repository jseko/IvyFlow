# 敏感数据检测规则

> v3.2 新增：事前防御层 — PII（个人身份信息）检测规则。
> 在 P0 阶段执行，检测用户需求输入中是否包含敏感数据。

---

## 检测规则

### 身份证号码

```
正则：/\b[1-9]\d{5}(18|19|20)?\d{2}(0[1-9]|1[0-2])(0[1-9]|[12]\d|3[01])\d{3}[\dXx]\b/
说明：匹配 18 位或 15 位中国大陆身份证号码
命中行为：提示用户脱敏（建议替换为 `ID_NUMBER_PLACEHOLDER`）
```

### 手机号码

```
正则：/\b1[3-9]\d{9}\b/
说明：匹配中国大陆 11 位手机号码
命中行为：提示用户脱敏（建议替换为 `PHONE_PLACEHOLDER`）
```

### 银行卡号

```
正则：/\b\d{13,19}\b/
说明：匹配 13-19 位银行卡号（Luhn 校验不做）
命中行为：提示用户脱敏（建议替换为 `BANK_CARD_PLACEHOLDER`）
```

### 邮箱地址

```
正则：/\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b/
说明：匹配标准邮箱地址格式
命中行为：提示用户确认是否为测试数据，生产数据须脱敏
```

---

## 检测逻辑

```
P0 阶段 / 步骤一需求解析时：
    │
    ├── 扫描用户输入的所有文本（需求描述、参数示例、配置项）
    │
    ├── 逐条运行 4 组正则
    │
    ├── 命中 → 收集命中项类型 + 位置
    │   ├── 身份证/手机号/银行卡 → 高风险，建议拒绝或强制脱敏
    │   └── 邮箱 → 中风险，提示确认是否为测试数据
    │
    └── 输出检测报告：
        通过 / 命中 X 项 → 提示脱敏后继续
```

---

## 处理方式

| 风险等级 | 数据类型 | 处理方式 |
|---------|---------|---------|
| 高 | 身份证/手机号/银行卡 | 拒绝执行，直到用户提供脱敏后的需求 |
| 中 | 邮箱 | 提示用户，用户确认后可继续 |
| 低 | 地址/姓名 | 提示注意，不阻断 |

---

## 注意事项

- 此规则为静态正则检测，不保证 100% 精确
- 假阳性（如需求文档中的示例手机号 `13800138000`）由用户判断
- 脱敏替换值建议使用全大写占位符，便于后续搜索和替换
