# Code Review Checklist

通用代码审查检查清单，适用于 Java / TypeScript / Vue 等项目主要语言。

## 功能正确性

- [ ] 代码是否实现了需求描述的功能
- [ ] 边界条件是否处理（空值、零值、最大值、异常输入）
- [ ] 错误路径是否正确处理（异常捕获、错误返回、降级逻辑）
- [ ] 是否有遗漏的逻辑分支

## 代码质量

- [ ] 命名是否清晰一致（变量、方法、类名遵循项目约定）
- [ ] 是否有重复代码（应提取公共方法/组件）
- [ ] 方法/函数长度是否合理（建议 < 30 行）
- [ ] 是否有未使用的变量、方法、import
- [ ] 注释是否必要且准确（避免无意义注释，保持与代码一致）

## 安全性

- [ ] 用户输入是否校验（类型、长度、格式、范围）
- [ ] SQL 是否使用参数化查询（禁止拼接）
- [ ] 敏感数据是否加密存储和脱敏展示
- [ ] 权限校验是否到位（接口级 + 数据范围级）
- [ ] 是否有信息泄露风险（日志、错误消息、响应体）

## 性能

- [ ] 是否有不必要的数据库查询（N+1 问题）
- [ ] 缓存是否合理使用（热点数据、TTL、失效机制）
- [ ] 大数据量操作是否分页/流式处理
- [ ] 是否有阻塞操作放在了主线程

## 可维护性

- [ ] 是否遵循 SOLID 原则（单一职责、开闭原则）
- [ ] 是否遵循项目分层约定（Controller → Service → Mapper）
- [ ] 新增代码是否与现有代码风格一致
- [ ] 是否有过度设计（YAGNI：只实现当前需要的）
- [ ] 配置是否外部化（硬编码值应提取为配置项）

## 测试

- [ ] 是否有对应的单元测试
- [ ] 测试是否覆盖了正常路径和异常路径
- [ ] 测试是否独立（不依赖其他测试的执行顺序）
- [ ] Mock 是否合理（不 Mock 被测类本身的方法）
