# QA Phase 1: 用例设计

## 目标

基于 PRD 和设计文档，生成覆盖全面的测试用例矩阵。

## 步骤

### 0. 入口状态验证

```bash
ivy state show
```

验证当前阶段为 testcase。读取 `docs/prd/<feature-name>.md`。

### 1. 分析测试范围

从 PRD 中提取测试范围：

- 核心功能点（每个验收标准对应一个测试用例组）
- 边界条件（输入范围、并发限制）
- 异常场景（网络异常、权限不足、数据异常）

### 2. 生成测试用例

使用 `templates/testcase.md` 模板，按四类场景生成：

#### 正向场景

| ID | 场景 | 前置条件 | 步骤 | 预期 | 优先级 |
|----|------|---------|------|------|--------|
| TC-001 | 正常流程 | 已登录 | 操作-确认 | 成功 | P0 |

#### 逆向场景（3 个以上）

| ID | 场景 | 前置条件 | 步骤 | 预期 | 优先级 |
|----|------|---------|------|------|--------|
| TC-002 | 必填为空 | 已登录 | 不填-提交 | 错误提示 | P1 |
| TC-003 | 格式错误 | 已登录 | 错误格式-提交 | 格式校验 | P1 |
| TC-004 | 权限不足 | 未登录 | 直接访问 | 跳转登录 | P1 |

#### 边界场景

| ID | 场景 | 前置条件 | 步骤 | 预期 | 优先级 |
|----|------|---------|------|------|--------|
| TC-005 | 最大值 | 已登录 | 输入最大值 | 正常处理 | P1 |
| TC-006 | 空值 | 已登录 | 输入空值 | 正常处理 | P2 |

#### 性能场景

| ID | 场景 | 前置条件 | 步骤 | 预期 | 优先级 |
|----|------|---------|------|------|--------|
| TC-007 | 并发操作 | 多用户 | 同时操作 | 无数据冲突 | P2 |

### 3. 用例质量自检

使用 `prompt/testcase-checklist.md` 逐项检查。

### 4. 产出物

写入 `docs/testcases/<feature-name>.md`。

### 5. 守卫检查

```bash
ivy guard testcase --apply
```
