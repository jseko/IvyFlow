# QA Phase 3: 缺陷跟踪

## 目标

对测试执行中发现的缺陷进行规范化管理和跟踪。

## 步骤

### 0. 入口状态验证

```bash
ivy state show
```

验证当前阶段为 bug。读取 `test-results/<feature-name>.json`。

### 1. Bug 报告模板

每个缺陷按以下格式报告：

```markdown
## Bug: <简述>

**严重等级**: P0 / P1 / P2 / P3
**模块**: <模块名>
**发现版本**: <版本号>

### 复现步骤
1. 打开 <页面>
2. 点击 <按钮>
3. 输入 <数据>

### 预期结果
<描述预期行为>

### 实际结果
<描述实际行为>

### 环境信息
- 浏览器: Chrome 120
- 操作系统: macOS 14
- 屏幕截图: <附件>
```

### 2. 严重等级判定

| 等级 | 定义 | 示例 |
|------|------|------|
| P0 | 阻断性，核心功能不可用 | 无法登录、支付失败 |
| P1 | 严重，主要功能异常 | 数据丢失、权限绕过 |
| P2 | 一般，次要功能异常 | UI 错位、文案错误 |
| P3 | 轻微，体验问题 | 加载慢、提示不友好 |

### 3. 指派与跟踪

- P0/P1：立即通知开发者，标记为 BLOCKED
- P2：创建 Issue，指定迭代修复
- P3：记录到 backlog，后续优化

### 4. 产出物

写入 `docs/bugs/<bug-id>.md`。

### 5. 守卫检查

```bash
ivy guard bug --apply
```
