# Context Engineering — Six Questions

Before beginning the design phase, verify the project context answers these six questions. If any answer is unclear, pause and clarify before proceeding.

## Q1: What does this project do?

The one-paragraph elevator pitch. What problem does it solve? Who are the users?

## Q2: What is the tech stack?

Language, framework, database, caching, message queue, deployment platform. Be specific with versions.

## Q3: What are the conventions?

Naming patterns, file organization, error handling style, logging approach, test patterns.

## Q4: What are the constraints?

Performance targets (latency, throughput), security requirements, compliance needs, resource limits.

## Q5: What are the anti-patterns?

Known pitfalls, deprecated approaches, "we tried X and it failed" lessons, tech debt hotspots.

## Q6: What is the deployment flow?

Build process, test pipeline, staging environments, release cadence, rollback procedure.
