# PM Phase 5: 验收

## 目标

对照 PRD 验收标准，逐项验证功能是否符合预期。

## 步骤

### 0. 入口状态验证

```bash
ivy state show
```

验证当前阶段为 accept。读取 `docs/prd/<feature-name>.md` 和 `docs/pm/review-record.md`。

### 1. 验收执行

逐项对照 PRD 中的验收标准：

| 验收项 | 预期 | 实际 | 状态 |
|--------|------|------|------|
| AC-001 | 项目创建成功 | 符合预期 | Pass |
| AC-002 | 提示错误信息 | 提示文案需优化 | Fail |
| AC-003 | 限制超长输入 | 符合预期 | Pass |

### 2. 异常场景验证

- [ ] 网络异常时的降级处理
- [ ] 权限不足时的拦截
- [ ] 数据异常时的容错

### 3. 用户体验验证

- [ ] 操作流程顺畅
- [ ] 错误提示友好
- [ ] 加载状态有反馈
- [ ] 空状态有引导

### 4. 验收决策

| 决策 | 条件 |
|------|------|
| 验收通过 | 所有 Must have 验收项通过 |
| 有条件通过 | Must have 通过，Should have 有少量问题 |
| 验收不通过 | Must have 有未通过项，回退修复 |

### 5. 用户确认（阻塞点）

展示验收结果，等待用户确认：

- 确认验收：进入归档
- 需要修复：列出问题，回退给开发者
- 暂缓验收：记录原因

### 6. 产出物

写入 `docs/pm/acceptance-report.md`。

### 7. 守卫检查

```bash
ivy guard accept --apply
```
