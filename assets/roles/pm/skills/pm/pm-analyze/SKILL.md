# PM Phase 2: 需求分析

## 目标

对收集的需求进行深度分析，产出结构化的需求分析报告。

## 步骤

### 0. 入口状态验证

```bash
ivy state show
```

验证当前阶段为 `analyze`。读取 `docs/pm/collect-summary.md`。

### 1. 竞品调研

如果涉及竞争功能：

- 选择 3-5 个直接竞品
- 对比核心功能实现方式
- 分析竞品的优势和不足
- 提炼可借鉴的最佳实践

### 2. 用户故事拆解

每个核心功能拆解为用户故事：

```
As a <角色>
I want <功能>
So that <价值>
```

示例：
```
As a 普通用户
I want 用手机号注册账号
So that 可以快速开始使用产品
```

### 3. 优先级排序

使用 MoSCoW + KANO 模型：

| 功能 | MoSCoW | KANO | 理由 |
|------|--------|------|------|
| 手机号注册 | Must | Basic | 核心流程 |
| 微信登录 | Should | Performance | 提升转化 |

### 4. 可行性评估

- **技术可行性**：现有技术栈是否支持
- **资源可行性**：团队能力和时间是否充足
- **业务可行性**：是否符合业务目标和 ROI

### 5. 产出物

写入 `docs/pm/analysis-report.md`：

```markdown
# 需求分析报告

## 竞品调研
| 竞品 | 功能 | 优势 | 不足 |

## 用户故事
| ID | 角色 | 功能 | 价值 | 优先级 |

## 可行性评估
| 维度 | 评估 | 风险 |
```

### 6. 守卫检查

```bash
ivy guard analyze --apply
```
