# PM Phase 1: 需求收集

## 目标

从多个来源收集和整理需求，为后续分析提供基础。

## 步骤

### 0. 入口状态验证

```bash
ivy state show
```

验证当前阶段为 `collect`。

### 1. 收集需求来源

从以下渠道收集需求信息：

- **用户反馈**：客服工单、用户访谈、NPS 反馈
- **数据分析**：用户行为数据、转化漏斗、留存率
- **竞品分析**：竞品功能对比、市场趋势
- **内部需求**：技术债务、性能优化、安全升级
- **业务需求**：业务方提出的诉求和 KPI

### 2. 分类汇总

按以下维度分类：

| 分类 | 示例 |
|------|------|
| 功能需求 | 新增功能、功能优化 |
| 体验优化 | 交互改进、UI 优化 |
| 性能需求 | 加载速度、响应时间 |
| 安全需求 | 权限控制、数据加密 |
| 技术需求 | 架构升级、技术债务 |

### 3. 优先级初筛

使用 MoSCoW 方法初步排序：
- **M**ust have：必须有的核心功能
- **S**hould have：应该有的重要功能
- **C**ould have：可以有的锦上添花
- **W**on't have：本期不做的功能

### 4. 产出物

写入 `docs/pm/collect-summary.md`：

```markdown
# 需求收集摘要

## 收集来源
- 来源1：N 条
- 来源2：N 条

## 分类汇总
| 分类 | 数量 | 优先级 |
|------|------|--------|

## 关键发现
- 发现1
- 发现2
```

### 5. 守卫检查

```bash
ivy guard collect --apply
```
