# PM Phase 4: 评审

## 目标

组织需求评审，确保 PRD 质量，生成验收用例。

## 步骤

### 0. 入口状态验证

```bash
ivy state show
```

验证当前阶段为 review。读取 `docs/prd/<feature-name>.md`。

### 1. 需求评审

使用 `prompt/review-checklist.md` 逐项检查：

#### 1.1 完整性检查

- [ ] 所有功能需求已覆盖
- [ ] 非功能需求已明确
- [ ] 边界条件已识别（3 个以上）

#### 1.2 清晰度检查

- [ ] 验收标准可测试
- [ ] 无歧义描述
- [ ] 技术术语已解释

#### 1.3 可行性检查

- [ ] 技术可行性已评估
- [ ] 资源可行性已评估
- [ ] 时间可行性已评估

### 2. 生成验收用例

基于 PRD 中的验收标准，生成详细的验收用例：

| 用例 ID | 场景 | 前置条件 | 步骤 | 预期结果 |
|---------|------|---------|------|---------|
| AC-001 | 正向-创建 | 用户已登录 | 点击新建-填写-提交 | 创建成功 |
| AC-002 | 异常-空值 | 用户已登录 | 不填必填项-提交 | 提示错误 |
| AC-003 | 边界-超长 | 用户已登录 | 输入超长文本-提交 | 提示限制 |

### 3. 变更影响分析

如果本次需求涉及已有功能的变更：

| 影响范围 | 描述 | 风险等级 |
|---------|------|---------|
| 模块 A | 接口变更 | 高 |
| 模块 B | UI 调整 | 低 |

### 4. 用户确认（阻塞点）

展示评审摘要，等待用户确认后再进入验收阶段。

### 5. 产出物

写入 `docs/pm/review-record.md`。

### 6. 守卫检查

```bash
ivy guard review --apply
```
