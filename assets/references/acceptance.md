# 验收标准编写规范

## Given/When/Then 格式

每条验收标准使用以下格式：

```
Given <前置条件>
When <触发动作>
Then <预期结果>
And <附加条件>
```

## 场景覆盖要求

- 正向场景：至少 1 个（正常流程）
- 逆向场景：至少 3 个（异常输入/权限不足/网络异常）
- 边界条件：至少 2 个（最大值/最小值/空值）

## 验收决策

| 决策 | 条件 |
|------|------|
| 验收通过 | 所有正向场景通过 |
| 有条件通过 | 正向通过，逆向有少量问题 |
| 验收不通过 | 正向有未通过项，回退修复 |
