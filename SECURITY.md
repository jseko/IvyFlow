# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in IvyFlow, please report it privately by opening a GitHub Issue with the label `security` and marking it as confidential (if your fork supports it).

**Do not** disclose the vulnerability publicly until it has been addressed.

## Response Time

We will acknowledge receipt within 48 hours and provide an estimated timeline for a fix.

## Scope

This security policy covers the IvyFlow CLI tool and its official distribution (npm package `ivyflow-cli`). Third-party integrations and plugins are outside the scope of this policy.
